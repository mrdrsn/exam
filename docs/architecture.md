# Архитектура проекта

Приложение реализовано с использованием архитектурного шаблона MVC.

## Model

Слой модели отвечает за представление данных предметной области:

- пациент;
- измерение;
- медицинские показатели;
- данные для отображения в интерфейсе.

## View

Слой представления отвечает за пользовательский интерфейс:

- главное меню;
- форма создания пациента;
- список пациентов;
- окно мониторинга;
- отображение текущих и предыдущих значений;
- отображение статистических показателей.

## Controller

Контроллер связывает интерфейс, модель и логику приложения.

Он отвечает за:

- обработку действий пользователя;
- создание пациента;
- открытие существующего пациента;
- запуск и остановку мониторинга;
- передачу новых измерений в интерфейс;
- запись данных в лог-файл.

## Генерация данных

Генерация медицинских показателей выполняется в отдельном потоке, чтобы пользовательский интерфейс продолжал работать во время мониторинга.

## Хранение данных

Для каждого пациента создается отдельный лог-файл.  
Файл используется для хранения истории измерений и загрузки последних значений при повторном открытии пациента.
