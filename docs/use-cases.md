# Сценарии использования

## UC-01. Создание нового пациента

**Актор:** пользователь  
**Предусловие:** пользователь находится в главном меню.

### Основной сценарий

1. Пользователь нажимает кнопку «Новый пациент».
2. Система открывает форму создания пациента.
3. Пользователь вводит ID пациента и ФИО.
4. Пользователь нажимает кнопку «Создать».
5. Система создает нового пациента.
6. Система создает лог-файл пациента.
7. Система открывает окно мониторинга пациента.

### Результат

Пациент создан и доступен для мониторинга.

---

## UC-02. Открытие существующего пациента

**Актор:** пользователь  
**Предусловие:** в системе есть ранее созданные пациенты.

### Основной сценарий

1. Пользователь нажимает кнопку «Открыть пациента».
2. Система отображает список доступных пациентов.
3. Пользователь выбирает пациента из списка.
4. Система загружает данные пациента из лог-файла.
5. Система отображает окно мониторинга пациента.

### Результат

Данные пациента загружены, пользователь может запустить мониторинг.

---

## UC-03. Запуск мониторинга

**Актор:** пользователь  
**Предусловие:** пациент выбран или создан.

### Основной сценарий

1. Пользователь нажимает кнопку «Начать мониторинг».
2. Система запускает генерацию данных в отдельном потоке.
3. Система генерирует значения температуры, сердечного ритма и ЦВД.
4. Система отображает текущие значения на экране.
5. Система сохраняет измерения в лог-файл пациента.
6. Система обновляет список последних 10 измерений.

### Результат

Мониторинг пациента запущен.

---

## UC-04. Обработка критического состояния

**Актор:** пользователь  
**Предусловие:** мониторинг пациента запущен.

### Основной сценарий

1. Система генерирует показатель вне нормального диапазона.
2. Система фиксирует время начала критического состояния.
3. Система выделяет проблемный показатель красным цветом.
4. Система показывает предупреждение пользователю.
5. После возвращения показателя в норму система фиксирует время восстановления.

### Результат

Критическое состояние зафиксировано и отображено пользователю.

---

## UC-05. Завершение мониторинга

**Актор:** пользователь  
**Предусловие:** мониторинг пациента запущен.

### Основной сценарий

1. Пользователь нажимает кнопку «Завершить мониторинг».
2. Система останавливает генерацию данных.
3. Последние полученные значения остаются отображенными в интерфейсе.

### Результат

Мониторинг пациента остановлен.
