# Функциональные требования

## Управление пациентами

FR-01. Система должна позволять создать нового пациента с указанием ID и ФИО.

FR-02. Система должна позволять открыть существующего пациента из списка.

FR-03. Система должна считывать список доступных пациентов из лог-файлов.

## Мониторинг показателей

FR-04. Система должна запускать генерацию данных после нажатия кнопки «Начать мониторинг».

FR-05. Система должна завершать генерацию данных после нажатия кнопки «Завершить мониторинг».

FR-06. Система должна генерировать данные в многопоточном режиме параллельно с работой пользовательского интерфейса.

FR-07. Система должна генерировать данные каждые 5 секунд.

FR-08. Система должна отслеживать температуру, сердечный ритм и центральное венозное давление.

## Отображение данных

FR-09. Система должна отображать ID и ФИО пациента.

FR-10. Система должна отображать последние 10 измерений температуры.

FR-11. Система должна отображать последние 10 измерений сердечного ритма.

FR-12. Система должна отображать последние 10 измерений центрального венозного давления.

FR-13. Если у пациента меньше 10 измерений, отсутствующие значения должны отображаться как «-».

## Критические состояния

FR-14. Система должна определять выход показателя за пределы нормального состояния.

FR-15. При выходе показателя за пределы нормы система должна отображать предупреждение.

FR-16. Показатель, вышедший за пределы нормы, должен выделяться красным цветом.

FR-17. Система должна фиксировать время начала критического состояния.

FR-18. После восстановления нормального состояния система должна фиксировать время восстановления пациента.

## Хранение данных

FR-19. Система должна хранить отдельный лог-файл для каждого пациента.

FR-20. Название лог-файла должно совпадать с ID пациента.

FR-21. При открытии существующего пациента система должна загружать последние 10 измерений из лог-файла.

## Статистика

FR-22. Система должна рассчитывать среднее значение для каждого показателя.

FR-23. Система должна рассчитывать математическое ожидание для каждого показателя.

FR-24. Система должна рассчитывать дисперсию для каждого показателя.

FR-25. Система должна рассчитывать первый и четвертый квартили.
